= API Walkthrough

Use the *Terminal* tab to interact directly with the CertChain REST APIs using `curl`.

== Get an Access Token

First, obtain a Keycloak token for the TechPulse organization:

[source,bash,subs="attributes"]
----
TOKEN=$(curl -sk -X POST \
  "{techpulse_keycloak_url}/realms/techpulse/protocol/openid-connect/token" \
  -d "grant_type=password" \
  -d "client_id=course-manager-ui" \
  -d "username=admin@techpulse.demo" \
  -d "password=admin" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
echo "Token: ${TOKEN:0:20}..."
----

== Issue a Certificate

Submit a certificate issuance request to the cert-admin-api:

[source,bash,subs="attributes"]
----
curl -sk -X POST \
  "{techpulse_cert_admin_api_url}/api/v1/certificates" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "certID": "API-TEST-001",
    "studentID": "student01@techpulse.demo",
    "studentName": "Jane Doe",
    "courseID": "FSWD-101",
    "courseName": "Full-Stack Web Dev",
    "issueDate": "2026-03-01",
    "expiryDate": "2028-12-31",
    "grade": "A",
    "degree": "Professional Certificate"
  }' | python3 -m json.tool
----

Expected: HTTP 200 with the full certificate JSON including `status: "ACTIVE"`.

== Verify a Certificate (Public)

Query the public verify-api — no authentication needed:

[source,bash,subs="attributes"]
----
curl -sk "{verify_api_url}/api/v1/verify/API-TEST-001" | python3 -m json.tool
----

The response includes status, student name, course, organization, and dates — but *not* grade or degree.

== Batch Verify Multiple Certificates

Verify several certificates in a single request:

[source,bash,subs="attributes"]
----
curl -sk "{verify_api_url}/api/v1/verify/batch?ids=API-TEST-001,FAKE-CERT-123" \
  | python3 -m json.tool
----

Each certificate returns its own status — VALID, REVOKED, or NOT_FOUND.

== View Student Transcript (Authenticated)

Get a student token and query the transcript endpoint:

[source,bash,subs="attributes"]
----
STUDENT_TOKEN=$(curl -sk -X POST \
  "{central_keycloak_url}/realms/certchain/protocol/openid-connect/token" \
  -d "grant_type=password" \
  -d "client_id=cert-portal" \
  -d "username=student01@techpulse.demo" \
  -d "password=student" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

# View all certificates for this student (includes grade/degree on owned certs)
curl -sk -H "Authorization: Bearer $STUDENT_TOKEN" \
  "{verify_api_url}/api/v1/transcript" | python3 -m json.tool
----

The transcript response includes *all certificates issued to this student's email* — with grade and degree visible on the student's own certificates.

== View a Single Certificate as Student

[source,bash,subs="attributes"]
----
curl -sk -H "Authorization: Bearer $STUDENT_TOKEN" \
  "{verify_api_url}/api/v1/transcript/API-TEST-001" | python3 -m json.tool
----

Compare this response to the anonymous verification — grade and degree are now visible.

== List Certificates (Admin)

List all certificates issued by TechPulse with pagination:

[source,bash,subs="attributes"]
----
curl -sk \
  "{techpulse_cert_admin_api_url}/api/v1/certificates?page=0&size=10" \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool
----

== Dashboard Stats (Admin)

Get aggregate certificate statistics:

[source,bash,subs="attributes"]
----
curl -sk \
  "{techpulse_cert_admin_api_url}/api/v1/dashboard/stats" \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool
----

Returns: `totalCertificates`, `activeCertificates`, `revokedCertificates`, `expiredCertificates`.

== Update a Certificate

Update mutable fields on an existing certificate:

[source,bash,subs="attributes"]
----
curl -sk -X PUT \
  "{techpulse_cert_admin_api_url}/api/v1/certificates/API-TEST-001" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "grade": "A+",
    "expiryDate": "2029-12-31"
  }' | python3 -m json.tool
----

== Revoke a Certificate

[source,bash,subs="attributes"]
----
curl -sk -X PUT \
  "{techpulse_cert_admin_api_url}/api/v1/certificates/API-TEST-001/revoke" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"reason": "Demo revocation test"}' | python3 -m json.tool
----

After revocation, re-verify to confirm the status changed:

[source,bash,subs="attributes"]
----
curl -sk "{verify_api_url}/api/v1/verify/API-TEST-001" | python3 -m json.tool
----

Expected: `status: "REVOKED"` with `revokedDate` and `revokedReason` fields.

== Admin Role Enforcement

Try accessing the cert-admin-api with a student token — it should be rejected:

[source,bash,subs="attributes"]
----
curl -sk -o /dev/null -w "HTTP %{http_code}" \
  "{techpulse_cert_admin_api_url}/api/v1/certificates" \
  -H "Authorization: Bearer $STUDENT_TOKEN"
----

Expected: HTTP 401 or 403 — students cannot access admin endpoints.

== Verify a Non-Existent Certificate

[source,bash,subs="attributes"]
----
curl -sk "{verify_api_url}/api/v1/verify/does-not-exist" | python3 -m json.tool
----

Returns HTTP 404 with `status: "NOT_FOUND"`.

== API Reference

=== cert-admin-api (per-org, requires `org-admin` role)

[cols="1,1,2"]
|===
| Method | Path | Description

| POST | `/api/v1/certificates` | Issue a new certificate
| GET | `/api/v1/certificates?page=0&size=20` | List org certificates (paginated)
| GET | `/api/v1/certificates/{certId}` | Get single certificate
| PUT | `/api/v1/certificates/{certId}` | Update mutable fields
| PUT | `/api/v1/certificates/{certId}/revoke` | Revoke (irreversible)
| GET | `/api/v1/courses` | List course catalog
| GET | `/api/v1/dashboard/stats` | Certificate stats
|===

=== verify-api (central, public + JWT)

[cols="1,1,1,2"]
|===
| Method | Path | Auth | Description

| GET | `/api/v1/verify/{certId}` | Public | Verify certificate status
| GET | `/api/v1/verify/{certId}/qr` | Public | Generate QR code (PNG)
| GET | `/api/v1/verify/batch?ids=X,Y` | Public | Batch verify
| GET | `/api/v1/transcript` | JWT | Student's certificates
| GET | `/api/v1/transcript/{certId}` | JWT | Single certificate
|===
