= Explore the Environment

== ArgoCD — GitOps Dashboard

CertChain is deployed via ArgoCD using the App-of-Apps pattern. The root Application generates 5 child Applications — one for central services, one per organization, and one for this lab guide.

Navigate to ArgoCD: {argocd_url}

Login with `admin` / `admin`, or click **Log in via OpenShift** if you're already authenticated.

You should see 6 ArgoCD Applications, all Synced and Healthy:

* *certchain* — Root App-of-Apps (generates the others)
* *certchain-central* — Fabric CA, orderers, cert-portal, verify-api, Keycloak, Grafana
* *certchain-techpulse* — TechPulse org (peer, orderer, CouchDB, cert-admin-api, course-manager-ui, Keycloak)
* *certchain-dataforge* — DataForge org (same layout)
* *certchain-neuralpath* — NeuralPath org (same layout)
* *certchain-showroom* — This lab guide

Click any Application to see its sync-wave ordering, resource tree, and health status.

== Grafana — Monitoring Dashboards

Navigate to Grafana: {grafana_url}

Grafana is configured with anonymous access — no login required.

=== Fabric Network Dashboard

Select *CertChain — Fabric Network* from the dashboard list.

This dashboard has 5 sections with 22 panels:

* *Overview* — Ledger height, BFT leader, total transactions, gossip known peers
* *Endorsement* — Proposal rate, endorsement duration p99, endorsement + ACL failures
* *Ledger & State Database* — Block commit latency, state DB commit latency, CouchDB query latency (all p99)
* *SmartBFT Consensus* — Committed block number, batch processing latency, transactions per batch, view decisions, pool size
* *Chaincode & gRPC* — Chaincode shim request duration, launch failures + timeouts, gRPC stream requests

=== Application APIs Dashboard

Select *CertChain — Application APIs*.

This dashboard has 5 sections with 18 panels:

* *Certificate Stats* — Total issued, revoked, verified, and not-found counters
* *Certificate Operations* — Operations rate, certificates issued per org (stacked)
* *cert-admin-api* — HTTP request rate, latency p99, HTTP 4xx/5xx errors, active connections
* *verify-api* — HTTP request rate, latency p99
* *JVM Runtime* — Heap memory, GC pause time, thread count

=== Infrastructure Health Dashboard

Select *CertChain — Infrastructure Health*.

This dashboard has 5 sections with 18 panels:

* *Status Overview* — Total pods, pods not ready, total restarts, namespaces
* *Blockchain Components* — CPU and memory for peers + orderers
* *Applications & Databases* — CPU and memory for APIs, UIs, CouchDB, and Postgres
* *Storage* — PVC usage percentage (gauge), PVC used vs capacity (bar chart)
* *Pod Health & Network* — Pod restart counts, pods not ready, network traffic by namespace

== Generate Traffic

To see the dashboards come alive, repeat the certificate issuance and verification steps from the previous sections. Each operation generates metrics that appear in the dashboards within 30 seconds.
